# `@field-iq/backend`

Fastify REST + WebSocket API. Node 22, Drizzle ORM, postgres-js, ioredis.

## Dev

```bash
pnpm --filter @field-iq/backend dev    # http://localhost:3000
```

Reads `.env.local` from the repo root via `src/config/env.ts`. The
critical secrets — `DATABASE_URL`, `REDIS_URL`, `JWT_SIGNING_SECRET` —
are generated by `pnpm run setup`.

## Migrations

```bash
pnpm --filter @field-iq/backend migrate    # apply pending migrations
pnpm --filter @field-iq/backend db:generate    # regenerate from schema.ts
```

## Sign-in (Prompt #10 — magic-link)

The clickable-email flow:

| Endpoint | Purpose |
| --- | --- |
| `POST /api/auth/request-link` | Email-only; always 204. Rate-limited 3/10 min per email. |
| `POST /api/auth/magic-link/request` | Back-compat alias; same handler. |
| `GET  /api/auth/verify?token=<uuid>` | Consumes the DB-backed token, mints a JWT, 302-redirects to the appropriate app origin with `#/auth/verify?session=<jwt>`. |
| `GET  /api/auth/me` | Bearer-authed; returns `{ user, org }` so the frontend can hydrate after the redirect. |
| `GET  /api/auth/config` | Public; reports `{ demoAuthEnabled }` so SignIn pages render the lab affordance only when appropriate. |
| `POST /api/auth/signout` | Returns `{ ok: true }`; no cookies set in v1. |
| `POST /api/auth/demo-login` | `admin@eon.ai` / `Demo1234!` shortcut. 404 unless `DEMO_AUTH_ENABLED=true`. |
| `POST /api/auth/magic-link/verify` | Stateless HMAC paste-token path. 404 unless `DEMO_AUTH_ENABLED=true`. |

### Env vars

| Var | Default | Notes |
| --- | --- | --- |
| `RESEND_API_KEY` | _unset_ | When unset, the email service logs the magic URL via Pino instead of sending. Posts directly to the Resend REST API — no SDK dep. |
| `EMAIL_FROM_ADDRESS` | `Field IQ <noreply@app.fieldiq.io>` | Used as the `From` header on Resend. |
| `MAGIC_LINK_BASE_URL` | `http://localhost:3000` | **Backend** host the email body links to. Must be reachable by the clicking browser. |
| `DASHBOARD_ORIGIN` | `http://localhost:3001` | Where the post-verify 302 sends non-technician roles. |
| `GLASSES_ORIGIN` | `http://localhost:3002` | Where the post-verify 302 sends technicians. |
| `DEMO_AUTH_ENABLED` | `false` | When true: paste-token + demo-password endpoints respond; sign-in card renders the paste-token form. |
| `DEFAULT_ORG_ID` | _unset_ | Override for auto-created users' org. When unset, the first row of `organizations` wins. |

### Data flow

```
Browser  →  POST /api/auth/request-link  →  rate-limit  →  insert magic_links  →  email
Email    →  GET  /api/auth/verify?token  →  consume row (used_at)  →  mint JWT  →  302
Browser  →  #/auth/verify?session=<jwt>  →  GET /api/auth/me        →  storeAuth
```

Tokens live in the `magic_links` table (UUID, 15 min TTL, single-use via
`used_at`, IP + UA logged). The stateless HMAC tokens (`auth/tokens.ts`)
still back the gated paste-token demo path; they have no DB footprint.

## Test

```bash
pnpm --filter @field-iq/backend test
```

the email service (dev stub + transport call + HTML escape), the rate
limiter (sliding-window math), the auth routes (config probe, request-link
always 204, demo gate, verify 400), URL builders, JWT roundtrip, and the
